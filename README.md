# Diplom_3
## Задание 3: Автотесты для UI сервиса [«Stellar Burgers»](https://stellarburgers.education-services.ru/)
___

### Реализованные сценарии

Созданы автотесты UI, покрывающие следующие сценарии:
1. Проверка основной функциональности:
* переход по клику на «Конструктор»;
* переход по клику на раздел «Лента заказов»;
* всплывающее окно с деталями при нажатии на ингредиент;
* закрытие всплывающего окна нажатием на крестик;
* увеличение счетчика ингредиента при добавлении ингредиента в заказ.

2. Раздел «Лента заказов»:
* увеличение счетчика «Выполнено за все время» при создании нового заказа;
* увеличение счетчика «Выполнено за сегодня» при создании нового заказа;
* отображение номера оформленного заказа в разделе «В работе».

### Структура проекта

- `allure_results` - пакет, содержащий результаты тестов Allure.
- `locators` - пакет, содержащий необходимые для тестирования локаторы, разделенные по страницам.
- `pages` - пакет, содержащий файлы с классами и методами для каждой страницы;
- `tests` - пакет, содержащий тесты, разделенные по страницам: 
-- `test_main_page.py` - тесты страницы с конструктором бургера;
-- `test_order_list_page.py` - тесты страницы с лентой заказов;
- `conftest.py` - содержит фикстуры для тестов.
- `data.py` - содержит тестовые данные.
- `helpers.py` - содержит вспомогательные функции, генерирующие рандомные тестовые данные, а также методы API для создания и удаления тестового пользователя.
- `README.md` - содержит описание проекта.
- `requirements.txt` - содержит зависимости проекта Python.
- `urls.py` - содержит URL-адреса и эндпоинты.

### Окружение

Тестирование проводилось в двух браузерах:
- Google Chrome 149.0.7827.115 (64 бит);
- FireFox 152.0 (64-разрядный).

### Запуск автотестов

**Установка зависимостей**

> `$ pip install -r requirements.txt`

**Запуск автотестов**

>  `$ pytest -v`

**Allure-отчет о тестировании в формате веб-страницы**

>  `allure serve allure_results`
